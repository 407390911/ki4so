简约互联网单点登录系统是一个简单、无状态、易扩展、易伸缩，特别适合大型互联网应用的单点登录解决方案。 如何保证安全 密码学 Hash算法 作用：防止数据篡改 对称密钥算法 DES, AES, … 作用：用于Credential 加密解密 非对称密钥算法 RSA 作用：用于各个应用和key server 之间获取密钥 SSO Key Server 统一管理各个注册应用的密钥 各个应用向SSO Key Server 获取自己的密钥，获取过程使用公钥私钥算法加密通信 各应用独立密钥 每个应用系统拥有一个唯一的AppId和一个与其对应的Credential密钥。 不同的应用系统使用不同的Credential密钥，目的是防止截取或伪造单点登录系统的Credential，并防止两个SSO应用系统之间互相截取或伪造对方的Credential。 密钥有一定的有效期 客户端安全： Httponly https

单点登录统一退出

各个应用的登录和退出地址由SSO service center统一管理

捆绑式登录 比如对于某两个应用站点，用户很大可能性会相互访问，那么在登录任一个应用成功之后，根据sso sevice center的配置，使用ajax 登录另一个站点。提升用户体验。 统一退出 在某一个应用退出之后，跳转到sso portal退出，依次调用各个子应用的退出接口，就完成了统一退出。

可扩展性可用性

SSO Portal 无状态，可以方便的水平扩展和垂直扩展

SSO Service Center 提供用户信息接口和为SSO Portal Service服务，同样可以方便扩展。

SSO Key Server 同样高可用， 即使挂掉一定的时间，各个应用还可以使用缓存的密钥验证用户信息，不会导致系统不可用。
